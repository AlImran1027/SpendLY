# Spendly — Smart Expense Tracking

> Scan a receipt. Track your spending. Split the bill.

Spendly is a cross-platform Flutter app that turns receipt photos into structured expense records using AI. It stores all data in Firebase Firestore, tracks budgets, visualises spending with custom charts, and lets you split restaurant bills with friends.

---

## Features

### Receipt Scanning & AI Extraction
- Capture receipts with the camera or pick from the gallery
- AI extracts merchant, date, total, line items, category, and payment method
- **Dual AI backend** — Google Gemini (cloud) with automatic fallback to LM Studio (local/private)
- Per-field confidence scores shown on the extraction results screen
- Editable before saving — correct anything the AI got wrong

### Expense Management
- Add expenses manually or from scanned receipts
- 8 categories: Groceries, Food/Restaurant, Medicine, Clothes, Hardware, Cosmetics, Entertainment, Others
- Line-item breakdown (quantity × price) per expense
- Receipt image stored with each record
- Full edit and delete from the detail screen

### Budget Tracking
- Set monthly budgets per category
- Animated progress bars with colour-coded health status (On Track / Near Limit / Over Budget)
- OS-level push notifications when a category exceeds 80 % or 100 % of its budget
- Budget data is fully persistent in Firestore

### Analytics Dashboard
- **Period selector** — Week / Month / Year
- **Spending Trend** bar chart — custom painter, peak bar highlighted, animated on load
- **By Category** donut chart with legend
- **Top Categories** ranked list with animated progress bars
- **Key Insights** — Peak Spend, Top Category, Transaction Count
- % change vs. previous period

### Split Bill
- Split any expense equally among multiple people
- Search friends by email address
- Sends a split request notification to each recipient
- Recipients can accept or reject; initiator sees the outcome
- All split activity synced via Firestore in real time

### Multi-Currency Support
14 currencies with a single tap — symbol and formatting update everywhere instantly:
BDT (৳), USD ($), EUR (€), GBP (£), INR (Rs.), JPY (¥), CAD (C$), AUD (A$), CHF (Fr.), CNY (¥), SAR (﷼), AED (د.إ), KWD (د.ك), SGD (S$)

### Other
- Firebase email/password authentication with password reset
- Push notifications via `flutter_local_notifications`
- Date format and language preferences
- Dark mode with full theme switching — all screens adapt automatically
- Staggered entrance animations throughout

## Getting Started

### Prerequisites

- Flutter SDK ≥ 3.11
- Dart SDK ≥ 3.11
- A Firebase project (free Spark plan is enough)
- Android Studio / Xcode (for device/emulator)

### 1. Clone & install dependencies

```bash
git clone https://github.com/AlImran1027/SpendLY.git
cd SpendLY
flutter pub get
```

### 2. Set up Firebase

This project requires Firebase for authentication and the database. Config files are git-ignored.

**a. Create a Firebase project**

Go to [console.firebase.google.com](https://console.firebase.google.com), create a project, and enable:
- **Authentication** → Sign-in method → Email/Password
- **Firestore Database** → Start in production mode

**b. Install the FlutterFire CLI**

```bash
dart pub global activate flutterfire_cli
```

**c. Run FlutterFire configure**

```bash
flutterfire configure
```

This generates `lib/firebase_options.dart`, `android/app/google-services.json`, and `ios/Runner/GoogleService-Info.plist` automatically.

**d. Set Firestore security rules**

In the Firebase Console → Firestore → **Rules** tab, paste the following and click **Publish**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Each user's expenses and budgets
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }

    // User profiles — readable by any authenticated user (split-bill search)
    match /userProfiles/{uid} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == uid;
    }

    // Split requests
    match /splitRequests/{splitId} {
      allow read: if request.auth != null && (
        resource.data.initiatorUid == request.auth.uid ||
        resource.data.recipientUids.hasAny([request.auth.uid])
      );
      allow create: if request.auth != null;
      allow update: if request.auth != null && (
        resource.data.initiatorUid == request.auth.uid ||
        resource.data.recipientUids.hasAny([request.auth.uid])
      );
    }

  }
}
```

> **Without these rules the app will show `permission-denied` errors** on expense loading, split-bill sending, and profile stats.

### 3. Configure AI extraction (optional but recommended)

Spendly supports two AI backends. You can configure them at runtime under **Profile → Settings**.

**Option A — Google Gemini (cloud, easiest)**

1. Go to [aistudio.google.com](https://aistudio.google.com) and create a free API key
2. In the app: Profile → Settings → **Gemini AI API Key** → paste your key

**Option B — LM Studio (local, private)**

1. Download [LM Studio](https://lmstudio.ai) and load a vision-capable model (e.g. LLaVA)
2. Start the Local Server inside LM Studio
3. In the app: Profile → Settings → **LM Studio** → enter your server URL (e.g. `http://192.168.1.x:1234`)

Gemini is used first if an API key is set; it falls back to LM Studio on connection failure.

### 4. Run the app

```bash
flutter run                  # Run on connected device / emulator
flutter run -d chrome        # Run as a web app
```

## Firestore Data Model

```
users/{uid}/
  expenses/{expenseId}        — merchant, category, amount, date, items[], image path
  budgets/{budgetId}          — category, year, month, amount

userProfiles/{uid}            — uid, email, displayName (for split-bill search)

splitRequests/{splitId}       — initiatorUid, recipientUids[], perPersonAmount,
                                merchant, date, recipients map (uid → status)
```

## AI Extraction Flow

```
Camera / Gallery
      ↓
ReceiptCaptureScreen  →  ReceiptPreviewScreen  →  ExtractionResultsScreen
                                                         ↓
                                               GeminiService.extractFromImage()
                                               (fallback: LMStudioService)
                                                         ↓
                                               ExtractedReceiptData
                                               (merchant, date, amount, items,
                                                category, payment, confidence)
                                                         ↓
                                              ExpenseEntryScreen  →  Firestore
```
